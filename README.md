# 現金経費管理アプリ

「現金版の通帳アプリ」。引き出した現金がどこへ消えたかを履歴で説明できることに特化した、経営者向けのシンプルなアプリです。

**1つのソース（`www/`）から、iOSアプリとWeb（PWA）の両方で使えます。**

| 形 | 説明 | 手順 |
|----|------|------|
| 📱 **iOSアプリ**（目標） | Xcodeで開いてiPhoneにインストール／TestFlight／App Store公開 | **[iOSアプリ化_手順.md](./iOSアプリ化_手順.md)** |
| 🌐 **Web / PWA** | `www/` を静的ホスティング。スマホで「ホーム画面に追加」 | 下記参照 |

---

## できること

| 段階 | 機能 |
|------|------|
| 基本 | 現金引き出しの記録（日付・金額） |
| 基本 | 経費使用の記録（日付・金額・分類・一言メモ・領収書写真） |
| 基本 | 今月サマリー（引き出した額／使った額／手元に残るはず） |
| 基本 | 月別の履歴一覧（◀ 月切替 ▶・通帳形式の集計） |
| 整合性 | 実際の手元現金を入れて「過不足」を確認 |
| 整合性 | 月末残高・翌月繰越の自動計算 |
| 分析 | 分類別の使用割合（円グラフ）・月別の使用推移（棒グラフ） |
| 管理 | 過不足から代表者貸付金の方向を提示 |
| 管理 | 領収書写真の保存・拡大表示 |
| 管理 | CSV書き出し・印刷/PDF保存・バックアップ（iOSでは共有シート経由） |

## 📱 iOSアプリとして使う（目標）

詳しい手順は **[iOSアプリ化_手順.md](./iOSアプリ化_手順.md)** にまとめています。要点だけ：

```bash
cd 現金経費管理アプリ
npm install        # 初回のみ（依存の取得）
npx cap sync ios   # www/ の変更をiOSへ反映
npx cap open ios   # Xcodeで開く
```

Xcodeで **Signing & Capabilities → Team** に自分のApple IDを設定し、iPhoneを繋いで **▶︎ Run** すればインストールされます。
（アプリ名：現金経費管理／Bundle ID：jp.sjdesign.cashbook／iOS 15以上）

## 🌐 Web / PWA として使う

### スマホでホーム画面に追加
- **iPhone（Safari）**：URLを開く → 共有ボタン → 「ホーム画面に追加」
- **Android（Chrome）**：URLを開く → ⋮ → 「アプリをインストール」

### 公開（ホスティング）
`www/` フォルダの中身を、そのまま静的サイトとして置けば動きます（ビルド不要）。
さくら / エックスサーバー / Netlify / Vercel / GitHub Pages など。
PWA（オフライン・ホーム画面追加）は **https** 配信時に有効です。

### 手元で試す（Mac）
```bash
cd 現金経費管理アプリ/www
python3 -m http.server 8000
# ブラウザで http://localhost:8000/ を開く
```

## データについて（大切）

- データは**この端末の中だけ**（iOSアプリ／ブラウザの IndexedDB）に保存されます。サーバーには送られません。
- そのため、**機種変更・アプリ削除・ブラウザのデータ削除で消えます。**
- 設定 ⚙ →「バックアップを保存」で、**ときどきバックアップ**してください（写真も含めて1つのファイルになります）。
- 新しい端末では「バックアップを読み込む」で復元できます。

## 設計の指針（仕様メモより）

- **書き込みは浅く。** 用途は「分類＋一言メモ」まで。
- **入力は速く。** 1件10秒。
- **見せ方を重視。** 数字を大きく、ひと目で。税務署に説明しやすく。
- **本人が使う前提。** 複数人運用は想定しない。
- **現金に特化。** カード・口座連携はしない。

> あえて機能を絞ることが、税務署への説明力につながる。
